# Question 4 — CLI: Navigation and Predicting Output

**Format:** Commands + predict output

---

> Assume a Linux/macOS/WSL terminal. Your home directory is `/home/user`.

You start in `/home/user`. The folder structure is:

```
/home/user
└── projects
    └── sctp
        └── data.csv
```


**(a)** Write the **single command** to move directly from `/home/user` into the `sctp` folder (where `data.csv` lives).

**Answer:**
_(cd /home/user/sctp)_

---

**(b)** You are now inside `sctp`. You run `pwd` and then `ls`. Write the **exact output** you expect from each.

**Answer:**
_()_


```text
$ pwd
# your predicted output: pwd /home/user/sctp

$ ls
# your predicted output: data.csv
```

---

**(c)** From inside `sctp`, write the command(s) to:
1. create a new folder called `output`, and
2. move back up **one** level into `projects`.

**Answer:**
_(mkdir output
cd ..)_

---

**(d)** You try to run `cd reports` and get `cd: no such file or directory: reports`. In one sentence, explain what this error tells you and one command you would run to figure out what *is* in the current folder.

**Answer:**
_(The error means that there is no directory named reports in the current location (or the name was typed incorrectly); I would run ls to see what files and folders are actually present in the current directory.)_
